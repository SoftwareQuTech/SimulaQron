setVals(remote_ip, remote_port, num_pairs, min_fidelity, max_time, priority, store, measure_directly)[source]¶

Stores required parameters of Entanglement Generation Protocol Request

Parameters:
  • remote_ip – int IP of the other node we are attempting to generate entanglement with
  • remote_port – int Port number of other node.
  • num_pairs – int The number of entangled pairs we are trying to generate
  • min_fidelity – float The minimum acceptable fidelity for the pairs we are generating
  • max_time – float The maximum amount of time we are permitted to take when generating the pairs
  • priority – obj Priority on the request
  • store – bool Specifies whether entangled qubits should be stored within a storage qubit or left within the communication qubit
  • measure_directly – bool Specifies whether to measure the communication qubit directly after the photon is emitted

This may be considered the initializer of the protocol, because it is called when the connection is completed. For clients, this is called once the connection to the server has been established; for servers, this is called after an accept() call stops blocking and a socket has been received.
